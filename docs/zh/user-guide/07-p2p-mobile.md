# 第七部分：P2P 共享与移动端

## P2P 终端共享

通过 QR 码将桌面终端共享给移动设备 — 无需云服务器。

### 工作原理

GGTerm 使用 iroh（QUIC + NAT 穿透）进行直接点对点连接：
- 90%+ P2P 直连成功率
- 针对 NAT 困难环境自动回退到中继
- 零运营成本（iroh 公共中继免费）
- Ticket：约 130 字符的 base32 字符串，可放入一个 QR 码

### 桌面端（主机）

1. 按 `Ctrl+Shift+Alt+Q` 打开共享覆盖层
2. 显示包含连接 ticket 的 QR 码
3. 用移动端应用扫描 QR 码（或复制 ticket 字符串）
4. 连接成功后，移动设备镜像你的终端
5. 按 `Esc` 或 `Ctrl+Shift+Alt+Q` 关闭共享

覆盖层显示：
- QR 码（深色模块渲染为矩形）
- 连接状态（等待中 / 已连接）
- Ticket 字符串（用于手动输入）
- 操作说明

### 数据流

- **PTY 输出 → 移动端**：所有终端输出同步到已连接的移动设备
- **移动端输入 → PTY**：移动端键盘输入转发到桌面 PTY
- **调整大小**：终端尺寸变更会同步传播
- **移动端本地回显**：移动端可立即看到输入的字符（无需等待 PTY 回显）

### 移动端（客户端）

#### 连接选项

| 选项 | 说明 |
|--------|-------------|
| SSH | 连接到远程服务器（主机、端口、用户、密码） |
| Echo Test | 诊断 — 回显输入的字符（无需服务器） |
| Scan QR | 通过 QR 码 P2P 连接桌面终端 |
| Share Terminal | P2P 主机模式（仅 Android — 需要本地 shell） |

#### 扫码流程

1. 在连接界面点击 **Scan QR**
2. 将摄像头对准桌面 QR 码
3. 终端输出显示在移动端
4. 在移动端键盘输入发送命令

#### iOS 与 Android

- **iOS**：仅支持 SSH + P2P 客户端（Scan QR）— 无本地终端
- **Android**：所有功能，包括本地 shell + P2P 主机

### 安全性

- P2P 连接加密（QUIC/TLS）
- SSH 服务器密钥指纹已记录（SHA256:base64 格式）
- SSH 同时支持密码和公钥认证

## SSH 连接管理器

存储和管理 SSH 连接：

通过命令面板：
- `ssh.manager` — 打开 SSH 连接管理器
- `terminal.import_ssh` — 从 `~/.ssh/config` 导入主机

功能：
- 主机条目包含名称、主机、端口、用户、认证方式
- TOML 持久化
- 模糊搜索
- 快速连接

## 本地 Shell（仅 Android）

安装了 Termux 或类似应用的 Android 设备可以在 GGTerm 移动端直接运行本地 shell。
