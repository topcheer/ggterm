# Part 5: AI 어시스턴트

## AI 기능

GGTerm은 터미널 출력을 설명하고, 명령을 제안하며, 자연어를 shell 명령으로 변환하는 AI 어시스턴트를 통합하고 있어요.

### AI 단축키

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+E` | 현재 출력 설명 |
| `Ctrl+Shift+S` | 다음 명령 제안 |
| `Ctrl+Shift+H` | 도움말 — 터미널에 대해 질문 |
| `Ctrl+Shift+N` | 자연어를 명령으로 변환 |
| `Esc` | AI 오버레이 닫기 |
| `Tab` (AI 오버레이에서) | 제안된 명령을 터미널에 삽입 |
| `Ctrl+Enter` (AI 오버레이에서) | 제안된 명령 즉시 실행 |

### AI 설정

```toml
[ai]
enabled = true
api_endpoint = "https://api.openai.com/v1"
model = "gpt-4"
```

AI 엔진은 OpenAI 호환 API 클라이언트를 사용하므로, 호환되는 엔드포인트는 모두 작동해요.

### AI 컨텍스트

AI 기능을 실행할 때, GGTerm은 다음에서 컨텍스트를 구성해요:
- 현재 터미널 출력 (화면에 보이는 부분)
- 현재 작업 디렉토리 (OSC 7)
- 실행 중인 명령 (OSC 133 마크)
- 종료 코드 (OSC 133 D 마크)

### Command Palette AI 명령

Command Palette(`Ctrl+Shift+P`)를 통해:
- `ai.explain` — 출력 설명
- `ai.suggest` — 명령 제안
- `ai.help` — 일반 도움말

## Shell 전환기

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+L` | 빠른 shell 전환기 |

설치된 shell(bash, zsh, fish 등) 중에서 선택하는 드롭다운을 열어요. 현재 pane에서 전환돼요.

## 명령 히스토리 사이드바

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+Y` | 명령 히스토리 사이드바 토글 |

기능:
- 현재 세션에서 실행한 명령의 전체 히스토리
- OSC 133 마크에서 동기화
- 종료 코드 표시 (초록/빨강 표시)
- 명령을 클릭하여 재실행

## 명령 탐색

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+Up/Down` | 명령 블록 사이를 탐색 |

OSC 133 마크를 사용하여 이전 명령 출력 블록 사이를 이동해요.

## 스니펫

스니펫 관리자는 자주 사용하는 명령을 저장해요:
- Command Palette를 통해 CRUD
- TOML 영속성
- 플레이스홀더 채우기 (예: `$USER`, `$HOST`)
- Command Palette를 통해 접근

## Broadcast 입력

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+Alt+B` | broadcast 모드 순환 |

세 가지 모드:
1. **None** — 활성 pane에만 입력 (기본값)
2. **AllPanes** — 활성 탭의 모든 pane에 입력 전송
3. **AllTabs** — 모든 탭의 활성 pane에 입력 전송

활성화되면 상태 표시줄에 `BCAST:AllPanes` 또는 `BCAST:AllTabs`가 표시돼요.

Command Palette를 통한 추가 broadcast 명령:
- 모든 pane에 `Ctrl+C` 전송
- 모든 터미널 재설정

## 세션 녹화

터미널 세션을 asciinema v2 형식으로 녹화해요:
- Command Palette를 통해 시작/중지
- 상태 표시줄에 `REC` 표시
- 출력: `.cast` 파일

## 워크스페이스

워크스페이스는 탭 그룹을 분리해요:

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+Alt+W` | 워크스페이스 순환 |

Command Palette를 통해:
- `workspace.next` — 다음 워크스페이스로 전환
- `workspace.prev` — 이전 워크스페이스로 전환
- `workspace.add` — 새 워크스페이스 생성

## 성능 모니터

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+G` | 성능 모니터 토글 |

FPS, 메모리, 셀 수, PID 정보를 표시하는 오버레이를 보여줘요.

## 사운드

| 단축키 | 동작 |
|----------|--------|
| `Ctrl+Shift+M` | 사운드 토글 |

활성화되면 터미널 bell(`\a`)이 소리로 재생돼요. 상태 표시줄에 `SND` 표시가 나타나요.

## 파일 미리보기

터미널 위로 파일을 드래그하면 미리보기 카드가 나타나요:
- 파일 아이콘 (카테고리별: 코드, 이미지, 아카이브 등)
- 파일 이름과 크기
- 카테고리별 색상

## 색상 선택기

ANSI 색상 시퀀스 위에 마우스를 올리면 hex 값을 보여주는 색상 견본이 나타나요.

## 컨텍스트 메뉴

터미널 콘텐츠 영역에서 우클릭:

| 동작 | 설명 |
|--------|-------------|
| 복사 | 선택 영역 복사 |
| 붙여넣기 | 클립보드에서 붙여넣기 |
| 모두 선택 | 모든 텍스트 선택 |
| 검색 | 검색 바 열기 |
| 지우기 | 화면 + 스크롤백 지우기 |
| 재설정 | 터미널 재설정 (RIS) |
| 수평 분할 | 좌/우 분할 |
| 수직 분할 | 상/하 분할 |

## 알림

- **데스크톱 알림**: OSC 9 (iTerm2) 및 OSC 777 (urxvt) 프로토콜
- **진행률 보고**: OSC 9;4가 상태 표시줄에 백분율을 표시해요
- **Bell**: 시각적 플래시 + 선택적 사운드
