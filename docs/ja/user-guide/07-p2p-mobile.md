# Part 7: P2P 共有とモバイル

## P2P ターミナル共有

QR コード経由でデスクトップのターミナルをモバイルデバイスと共有します — クラウドサーバーは不要です。

### 仕組み

GGTerm は直接 P2P 接続のために iroh（QUIC + NAT トラバーサル）を使用します：
- 90% 以上の P2P 直接接続成功率
- 困難な NAT に対する自動リレーフォールバック
- 運用コストゼロ（iroh のパブリックリレーは無料）
- チケット：約130文字の base32 文字列、1つの QR コードに収まる

### デスクトップ（ホスト）

1. `Ctrl+Shift+Alt+Q` を押して共有オーバーレイを開く
2. 接続チケット付きの QR コードが表示されます
3. モバイルアプリで QR コードをスキャンします（またはチケット文字列をコピー）
4. 接続されると、モバイルデバイスにターミナルがミラーリングされます
5. `Esc` または `Ctrl+Shift+Alt+Q` で共有を終了します

オーバーレイの表示内容：
- QR コード（暗いモジュールは矩形としてレンダリング）
- 接続ステータス（待機中 / 接続済み）
- チケット文字列（手動入力用）
- 手順

### データフロー

- **PTY 出力 → モバイル**: すべてのターミナル出力が接続されたモバイルデバイスにティーされます
- **モバイル入力 → PTY**: モバイルのキーボード入力がデスクトップの PTY に転送されます
- **リサイズ**: ターミナル寸法の変更が伝播されます
- **モバイルでのローカルエコー**: モバイルでは入力文字が即座に表示されます（PTY エコーを待機しない）

### モバイル（クライアント）

#### 接続オプション

| オプション | 説明 |
|--------|-------------|
| SSH | リモートサーバーに接続（ホスト、ポート、ユーザー、パスワード） |
| Echo Test | 診断用 — 入力文字をエコーバック（サーバー不要） |
| Scan QR | QR コード経由でデスクトップターミナルに P2P 接続 |
| Share Terminal | P2P ホストモード（Android のみ — ローカルシェルが必要） |

#### Scan QR の流れ

1. 接続画面で **Scan QR** をタップ
2. デスクトップの QR コードにカメラを向ける
3. ターミナル出力がモバイルに表示されます
4. モバイルキーボードで入力して送信

#### iOS と Android

- **iOS**: SSH + P2P クライアント（Scan QR）のみ — ローカルターミナルなし
- **Android**: ローカルシェル + P2P ホストを含むすべての機能

### セキュリティ

- P2P 接続は暗号化されています（QUIC/TLS）
- SSH サーバーキーのフィンガープリントがログに記録されます（SHA256:base64 形式）
- SSH はパスワード認証と公開鍵認証の両方をサポートしています

## SSH 接続マネージャー

SSH 接続を保存および管理します：

Command Palette から：
- `ssh.manager` — SSH 接続マネージャーを開く
- `terminal.import_ssh` — `~/.ssh/config` からホストをインポート

機能：
- ホストエントリ（名前、ホスト、ポート、ユーザー、認証方式）
- TOML 永続化
- ファジー検索
- クイック接続

## ローカルシェル（Android のみ）

Termux などがインストールされた Android デバイスでは、GGTerm モバイルで直接ローカルシェルを実行できます。
